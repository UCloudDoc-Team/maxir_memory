# 服务管理

服务管理提供系统运行状态监控、用量统计和日志查询功能，帮助管理员掌握实例的健康状况和使用情况。

## 系统状态监控

在管理后台，选择**服务管理** > **系统状态**，查看各核心组件的运行状态：

| 组件 | 说明 |
|------|------|
| **API 服务** | 核心 API 服务的运行状态和响应延迟 |
| **记忆引擎** | 记忆处理和检索引擎的状态 |
| **文档处理** | 文档上传和索引处理队列状态 |
| **向量数据库** | 向量存储和检索服务状态 |
| **冲突检测引擎** | 记忆冲突检测服务状态 |

状态指示：
- 绿色：正常运行
- 黄色：性能降级，但服务可用
- 红色：服务异常，需要关注

## 用量统计

### 存储用量

在**服务管理** > **存储用量**，查看：

- 总存储配额和已用量
- 按团队分布的存储占用
- 存储用量增长趋势（近 30 天）
- 文件类型分布（文档、图片、视频等）

### API 调用统计

在**服务管理** > **API 统计**，查看：

- 总 API 调用次数（今日 / 本周 / 本月）
- 调用成功率和错误率
- 平均响应延迟
- 按接口类型的调用分布
- Top 调用用户和团队

### Token 消耗统计

在**服务管理** > **Token 统计**，查看：

- 总 Token 消耗（本月）
- 按模型的 Token 消耗分布
- 按用户/团队的 Token 消耗排行
- Token 消耗趋势图（近 30 天）

## 服务日志

### 系统日志

1. 在**服务管理** > **系统日志**，查看系统级别的运行日志。
2. 支持按日志级别（INFO / WARN / ERROR）、时间范围、关键词筛选。
3. 可下载日志文件用于故障排查或审计。

### 访问日志

1. 在**服务管理** > **访问日志**，查看所有 API 请求记录，包括：
   - 请求时间、来源 IP、用户信息
   - 请求接口和参数
   - 响应状态码和耗时

### 错误日志

1. 在**服务管理** > **错误日志**，查看所有异常和错误记录。
2. 每条错误记录包含错误类型、堆栈信息和发生时的上下文，便于问题定位。

## 告警配置

管理员可配置以下告警规则，当触发条件满足时，通过邮件或系统通知提醒：

| 告警类型 | 触发条件示例 |
|----------|-------------|
| 存储配额告警 | 存储用量超过配额的 80% |
| Token 配额告警 | 月度 Token 消耗超过配额的 80% |
| 服务异常告警 | 任何核心组件状态变为红色 |
| 错误率告警 | API 错误率超过 5% |
| 登录失败告警 | 同一 IP 或用户连续登录失败超过指定次数 |

**配置步骤：**
1. 在**服务管理** > **告警配置**，点击**添加告警规则**。
2. 选择告警类型，设置触发阈值。
3. 配置告警接收人（邮箱）。
4. 点击**保存**。

## 数据备份

MAXIR Memory 在 UCloud 平台上，数据存储在 UCloud 提供的可靠基础设施上，平台层面已提供基础的数据冗余保护。

如需额外的数据备份：
1. 通过**服务管理** > **数据导出**，定期导出全量数据备份（JSON 格式）。
2. 或使用 API 定期拉取数据，自行存储到外部备份系统。

> 建议建立定期备份机制，对于关键业务数据，每周至少备份一次。

## 重启服务

在极少数情况下，您可能需要重启某个服务组件（如处理队列堵塞）：

1. 在**系统状态**页，点击目标组件的**重启**按钮。
2. 确认操作，重启过程通常持续 1-2 分钟。
3. 重启期间，该组件相关的功能可能短暂不可用。

> **注意**：重启操作会影响正在进行的任务（如文档处理、记忆提取等），请在非高峰时段进行。
